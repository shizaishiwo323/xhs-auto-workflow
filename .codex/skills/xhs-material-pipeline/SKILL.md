# 小红书素材生成流程

默认用中文工作。目标是把用户给的素材文件夹整理成可发布、可自动发布的小红书素材包。

## 什么时候使用

用户提到以下任务时使用本 skill：

- 小红书素材、标题、正文、封面、配图、爆款分析、二创；
- 数据采集、数据分析、公开数据源、论文/作业找数据等垂类内容；
- 根据账号数据、历史发布记录、爆款参考生成新笔记；
- 生成每日素材包、`publish_manifest.json`、`merged_table.xlsx`、发布图片清单；
- 校验标题 20 字、正文 1000 字、图片 18 张、话题分离、违规风险；
- 拆分多宫格长图、加 `to be here` 水印。

## 核心原则
- 参考 `docs/优质内容的判断标准.md`：主旨清晰，所有画面/文字围绕同一主题；表达通俗，少用黑话；结构完整，风格统一；必须给读者“获得感”，包括信息满足或情绪价值。
- 参考 `docs/优质封面和标题的标准.md`：封面和标题负责信息流点击，优先满足用户利益点，可用趋势、预期、好奇、共鸣、痛点、审美等触发；标题准确反映内容，不夸大误导。
- 账号数据优先，外部爆款可以高保真学习标题公式、事实选择、场景设置、案例组织、图片素材思路、开头方式、正文骨架、段落节奏、价值呈现、共鸣写法、封面逻辑和评论触发方式；不是泛泛总结，而是拆成可复用写法。
- 小红书文案的目标不是“像 AI 很会总结”，而是“像一个有经验的普通人，把一个对自己有用的发现讲清楚”。
- 选题先过 HKR：H=好奇心/点击钩子，K=知识密度/可执行价值，R=情绪或身份共鸣。可发布选题至少满足两项；只占一项时先补角度、补案例或换成更具体的人群/场景。
- 同一主题可以连续生产多篇内容；历史发布记录用于避开重复表达，不用于直接否决新内容。发现同主题已发时，必须优先换角度继续创作，而不是停止。
- 只有同一 `publish_manifest.json`、同一发布 key、标题正文高度一致且图片组合基本一致时，才判定为同一篇重复内容并停止；停止原因必须写成“同一篇重复提交”，不能写成“同主题已发布”。
- 评论互动优先于点赞收藏：优先学习能引发真实问题、经验分享、A/B 选择、边界讨论的内容。
- 二创允许“贴着爆款写法模仿”，包括相近的标题句式、事实呈现、场景选择、案例逻辑、图片素材思路、正文推进顺序、段落长短、情绪拐点和结尾提问；关键是用自己的表达体系重新组织，而不是只做泛泛借鉴。
- 不复制原句、独特表达、作者身份设定、头像、账号名、私有数据或专有作品；对方的事实、场景和案例可以学习使用，甚至可以作为自己的亲历以及把对方经历写成自己的经历。
- 学习图片素材时，可以把爆款图片作为参考底图和真实数据来源，学习并保留其中有价值的事实信息、表格字段、代码片段、数据结果、场景元素、构图、主体、信息层级、画面顺序和色彩节奏。
- 基于参考图做发布图时，必须通过 [$imagegen](/Users/wangbin/.codex/skills/.system/imagegen/SKILL.md) 二次创作：重绘或重排主体、表格、代码、截图感元素、醒目文字、箭头标注、圈画重点或表情贴纸，生成自己的封面/配图。
- 图片二创必须去掉账号名、头像、水印、私信入口、二维码、联系方式和其他私有识别元素；不要以规避平台识别为目标，不直接搬运原图或水印截图。
- 只要素材来源里存在爆款图片、表格截图、代码截图、数据结果图或场景图，图片创作就不能退回到“只学构图”。必须选定参考图作为事实来源，保留其中可迁移的字段、代码、结果、画面顺序或场景主体，再用 imagegen 重绘/重排并加阅读指引。除非参考图质量差、含隐私或违规元素且无法脱敏，否则不能跳过这一环。
- 正文里不要写小红书话题标签；话题只放进 `recommended_topics`、`topics` 或 `tags` 等元数据字段。
- 不写抽奖、求评论送资料、私信领取、互关互赞、福利交换等表达；互动只能是内容相关、可自然回答的问题。
- 每次生成可发布素材包时，文案必须走 [$humanizer](/Users/wangbin/.codex/skills/humanizer/SKILL.md) 终稿润色；封面和配图只要需要新视觉，必须走 [$imagegen](/Users/wangbin/.codex/skills/.system/imagegen/SKILL.md) 生成，不允许用 PIL、matplotlib、HTML/CSS、SVG 或纯代码绘图替代 imagegen 作为最终发布图来源。
- 涉及塔罗牌时，只写牌面知识、牌阵学习、卡牌文化、美学故事、娱乐参考、自我觉察、情绪复盘、关系沟通启发；不要写预测未来、改命转运、代开光、付费占卜、收集生日八字手机号，也不要关联未成年人、考试和升学结果。


## 标准流程

1. **检查素材**
   - 用 `rg --files` 或 `find` 查看文件。
   - 优先找 Excel/CSV 中的 `标题`、`点赞`、`收藏`、`评论`、`推文`、`话题`、`媒体文件路径`、`封面地址` 等列。
   - 如果有账号数据，找 `笔记标题`、`浏览量`、`点赞量`、`收藏量`、`评论量`、`转发量`。
   - 如存在违规历史目录，必须先看：`/Users/wangbin/data/jupyter/实战/爬网页/小红书/自动化工作流/docs/目前遇到的违规情况`，提炼“历史违规规避清单”。
   - 如有社区规范 PDF，生成发布内容前先参考：`/Users/wangbin/data/jupyter/实战/爬网页/小红书/自动化工作流/docs/xhs_community_guidelines.pdf`。

3. **分析爆款参考**
   - 统一 `1.7万` 这类互动数为数字。
   - 推荐排序：`4*评论 + 2*收藏 + 点赞 + 2*转发`，有浏览量时再看评论率。
   - 只把“标题完整、正文完整、互动数据可读、媒体清楚”的笔记作为正文学习样本；只有标题没有正文的样本只能学习标题/封面入口，不能学习正文写法。
   - 如果 `posts_topN.xlsx`、`summary.csv` 或等价文件里高互动笔记的 `note_text`/`推文`/`正文` 大面积为空，必须先提醒用户“疑似风控或详情页正文未抓到”，说明完整参考不足；不足 3 条完整爆款参考时，不要假装已经学习正文结构。
   - 选 3-5 条评论质量高、媒体清楚、可迁移且正文完整的参考。
   - 对每条完整参考记录高保真拆解：标题钩子、标题句式、事实选择、场景设置、案例组织、第一句话、开头进入方式、正文推进顺序、段落节奏、具体价值点、共鸣句、结尾问题、封面布局、配图素材、配图节奏、公开数据源、评论触发点、可复用和必须避开的部分。
   - 二创时优先保留爆款中已验证有效的事实、场景、案例、画面元素和价值点，再用自己的表达规则重写标题、正文、封面文案和图片生成提示词；如果事实存在时效性或合规风险，先核实或弱化成观察。
   - 如果发布历史里已有同主题内容，先列出已发内容的标题、正文结构、封面钩子和图片顺序；新素材包至少替换 3 项，例如读者问题、切入角度、字段清单、流程步骤、案例场景、封面钩子、图片结构或素材来源。

4. **生成文案**
   - 标题给 3-8 个，直接发布的标题必须不超过 20 个中文字符。
   - 动笔前先写清楚 4 个内部判断：这篇给谁看、读者正在卡在哪里、读完能拿走什么、哪个具体场景/发现负责开头。判断不清楚时，先补素材，不要直接套模板。
   - 同主题续写时，标题不能只做同义改写；正文开头、主体结构、读者收获和结尾问题都要和已发布内容拉开差异。
   - 标题关键词要自然融入，可参考疑问式、数字、解决问题、情感连接、挑战/好奇、时令个性化、直接指令等公式；不要生硬堆词。
   - 正文必须不超过 1000 字，结构根据内容灵活调整。
   - 开头从具体场景、具体问题、具体发现或一句真实判断切入；不要写“随着时代发展”“很多人都知道”“今天给大家分享”这类空泛开头。
   - 正文优先采用“具体观察 -> 为什么会这样 -> 可执行做法/清单/案例 -> 坑点或边界 -> 内容相关问题”的短内容弧线。情绪类内容则用“具体瞬间 -> 当时的体感 -> 复盘出的动作 -> 给同处境读者的一句话”。
   - 每个核心观点至少要落到一个具体支撑：素材原句、评论问题、步骤、字段、数字、对比、失败边界、真实限制或可截图验证的信息。没有支撑的漂亮判断要删掉或改成疑问。
   - 写方法论时，每一段都要给一个今天能执行的动作，并坦诚说明适用边界、学习成本或容易失败的地方；不要只画“马上变好”的饼。
   - 写共鸣时，先替读者把处境说具体，再给建议。例如先承认“你不是不努力，可能只是卡在不知道从哪一步开始”，再给下一步。
   - 知识点要像聊天时顺手掏出来，围绕眼前素材解释；不要写成百科段落或培训课讲义。
   - 语言要保留一点真人的不完美，为了网感可以“家人们”“狠狠拿捏”“绝绝子”等网络用梗。
   - 节奏要适合手机滑读：短段落、长短句交替，关键判断可单独成段；轮播图能承接的步骤和清单不要全部塞进正文。
   - 写完正文后自查：主旨是否一眼能懂；是否有可复刻步骤、清单、案例、字段、模板或明确情绪价值；是否能让读者真的学到东西或获得共鸣。
   - 反 AI 味自查必须通过：没有宏大开头、没有“首先/其次/最后”机械分点、没有“综上所述/值得注意的是/不难发现/这意味着/本质上/赋能/闭环”等报告腔，没有空泛赞美，没有编造第一人称经历，不用“很稳”这种表达
   - 价值感自查必须通过：删掉任何只会说“很有用、很治愈、很适合普通人”但没有动作、例子、边界或判断标准的句子。
   - 正文、`推文`、`body` 字段都不能包含 `#话题`、话题块或话题占位。
   - 话题来自爆款分析或素材本身，单独存入 `recommended_topics` 等字段。
   - 合集名可写 `collection_name`，不超过 20 字；合集简介不超过 50 字。
   - 最后必须调用 [$humanizer](/Users/wangbin/.codex/skills/humanizer/SKILL.md) 做终稿润色
   - 润色后要把证据写入 `04_分析依据/humanizer润色记录.md`：包含润色前草稿摘要、humanizer 发现的 AI 感问题、HKR 判断、价值感自查、最终正文、确认正文无话题标签。
   - `publish_manifest.json` 必须写入 `humanizer_used: true`，并写入 `humanizer_record` 指向 `04_分析依据/humanizer润色记录.md`。

5. **生成或处理图片**
   - 需要新图时必须调用 [$imagegen](/Users/wangbin/.codex/skills/.system/imagegen/SKILL.md)，优先生成竖版手机图，建议 1080x1440、3:4；封面和每张配图都要一页一图生成，不把多页做成一张 contact sheet。
   - 如果素材包需要封面/配图，而用户没有提供可直接发布的原图，默认就是“需要新图”，必须走 imagegen；不得用本地 PIL、matplotlib、HTML/CSS、SVG、canvas 或截图式代码图当最终发布图。
   - 封面优先 3:4，保证双列信息流里曝光面积和可读性；封面必须和主题强相关，主体清楚，第一眼能看出利益点。
   - 封面可加关键词或看点，但文字放在空白区或主体边缘，不遮挡核心内容；系列内容保持风格一致。
   - 一页生成一张图，不把多页做成一张 contact sheet 当最终图。
   - 有参考图时，可把参考图里的真实数据、表格字段、代码片段、结果截图感、场景主体和画面顺序作为 imagegen 二创素材；加醒目标题、箭头、圈画、表情或短注释来强化阅读路径。
   - 二创图必须替换所有账号、头像、水印、私信入口、二维码、联系方式和专有元素；如果参考图含隐私、未脱敏用户数据或违规引导，不要保留。
   - 最终发布图里的中文内容应由 imagegen 直接生成。不要用 PIL、matplotlib、HTML/CSS、SVG 等本地代码给发布图叠正文，除非用户明确要求。
   - 本地代码只用于裁剪、压缩、拆图、搬运、加小水印、更新 manifest。
   - 每次 imagegen 调用后，必须检查输出是否有错字、内部流程词、违规诱导、路径、prompt、TODO、模型名或工具名；不合格就重生成或移出上传顺序。
   - 必须把 imagegen 证据写入 `04_分析依据/imagegen生成记录.md`：每张图的用途、最终 prompt 摘要、生成方式、选中原图路径、最终上传图路径、人工检查结果。
   - `imagegen生成记录.md` 逐图必须包含：参考图路径或跳过原因、保留的真实信息点（字段/代码/数据结果/场景主体/画面顺序）、二创动作（重绘/重排/箭头/圈画/贴纸/短注释等）、清理项（账号/头像/水印/二维码/联系方式/隐私）、最终上传图路径。
   - `publish_manifest.json` 必须写入 `imagegen_used: true`、`imagegen_record`、`image_generation_method`，并且 `images[].path` 指向 imagegen 输出经裁剪/压缩/水印后的最终上传图。
   - 发布图不能出现内部流程词：`校验`、`发布限制`、`素材包`、`manifest`、`publish_queue`、文件路径、prompt、TODO、模型名、工具名等。
   - 如果图片里出现内部说明、违规诱导、平台限制表或生产注释，必须移出上传顺序或重新生成。

6. **创建素材包**
   - 默认目录：
     ```text
     xhs每日生成素材/YYYY-MM-DD/选题名称/
     ├── 00_自动发推适配/
     ├── 01_文案/
     ├── 02_封面/
     ├── 03_配图/
     ├── 04_分析依据/
     └── README.md
     ```
   - 原图保留，新图另存，不覆盖源文件，除非用户明确要求。
   - 爆款参考、账号复盘、违规规避清单放入 `04_分析依据/`。
   - 同主题续写的新包目录名要体现差异化角度，例如 `抖音评论字段清单_可发布版`、`抖音账号数据复核_可发布版`，不要复用已成功提交的素材包目录和 manifest。

## 自动发布适配

生成或更新这些文件：

- `00_自动发推适配/merged_table.xlsx`
- `00_自动发推适配/publish_queue.xlsx`
- `00_自动发推适配/publish_manifest.json`
- `00_自动发推适配/发布图片清单.txt`

Excel 至少包含 `标题`、`推文`、`封面地址`。兼容旧表时保留 15 列：

`链接`, `标题`, `点赞`, `作者`, `作者链接`, `推文`, `收藏`, `评论`, `Unnamed: 8`, `封面地址`, `Unnamed: 10`, `Unnamed: 11`, `Unnamed: 12`, `Unnamed: 13`, `Unnamed: 14`

`publish_manifest.json` 中图片顺序为：封面第一，其余配图按上传顺序排列。`body` 和 `publish_row.推文` 必须无话题标签。

## 校验脚本

在 skill 目录下使用脚本，`python` 不可用时用 `python3`。

```bash
python3 scripts/validate_xhs_package.py <package-path>
```

检查标题、正文、图片数量、图片存在性、尺寸和疑似拼接图。

```bash
python3 scripts/split_carousel_images.py <package-or-image-path> \
  --report <package>/00_自动发推适配/配图裁剪检测报告.json
```

拆分横向拼接图，输出到 `03_配图/单张裁剪/`，并补到 1080x1440。

```bash
python3 scripts/watermark_images.py <input-folder-or-image> \
  --output-dir <package>/03_配图/加水印 \
  --text "to be here"
```

拆图后再加水印。最终 manifest 和图片清单必须指向单张、加水印后的最终上传图。

## 最终 QA

发布前检查：

- 标题 <= 20 字，正文 <= 1000 字，图片 <= 18 张；
- 爆款参考至少有 3 条同时包含标题、正文、互动数据和媒体线索；若不足，已明确告知用户参考不完整，且没有把缺正文样本用于正文写法学习；
- 已记录参考笔记的高保真写法拆解：标题句式、开头方式、正文骨架、段落节奏、价值点、共鸣句和结尾问题；
- 已完成 HKR 判断，至少满足两项；若只满足一项，已明确补角度、补素材或改选题；
- 正文有明确可带走价值，至少包含步骤、清单、模板、字段、案例、对比、避坑、判断标准或可复用动作中的一种；
- 正文开头是具体场景、具体问题、具体发现或真实判断，不是宏大背景和模板套话；
- 每个核心观点都有素材细节或可验证信息支撑，没有编造第一人称经历；
- 已完成反 AI 味自查，删除报告腔、空泛赞美、机械分点和只有概念没有细节的句子；
