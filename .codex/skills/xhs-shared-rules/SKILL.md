# 小红书共享规则

本文件只放跨流程共用的硬规则。具体怎么写文案、怎么生成素材包、怎么爬取、怎么发布，由对应 skill 负责。

## 适用范围

- 图文素材包生成：`xhs-material-pipeline`
- 自动化编排和发布：`xhs-auto-workflow-material-pipeline`
- 其他小红书相关维护任务需要复用通用约束时

## 项目与环境

- 默认中文沟通。
- 项目根目录：`/Users/wangbin/data/jupyter/实战/爬网页/小红书/自动化工作流`
- 项目 Python 默认使用 conda `base` 环境，优先用：`/Users/wangbin/anaconda3/bin/python`
- `archive/备份` 只作参考和回溯基线，默认不得修改。
- 禁止批量删除文件或目录。需要删除时，只能一次删除一个明确路径的文件；需要批量删除时停止并请用户手动处理。

## 内容硬规则

- 标题不超过 20 个中文字符。
- 正文、`推文`、`body` 等发布正文字段不超过 1000 字，且不能包含 `#话题`、话题块或话题占位。
- 话题只写入 `recommended_topics`、`topics`、`tags` 等独立元数据字段。
- 不写抽奖、求评论送资料、私信领取、互关互赞、福利交换等表达；互动问题必须和内容本身相关。
- 涉及品牌、产品功效、医疗、金融、法律等高风险内容时，不夸大效果，并提醒核实合规表述。
- 涉及塔罗牌时，只写牌面知识、牌阵学习、卡牌文化、美学故事、娱乐参考、自我觉察、情绪复盘、关系沟通启发；不写预测未来、改命转运、代开光、付费占卜、收集生日八字手机号，也不关联未成年人、考试和升学结果。

## Humanizer 规则

- 每个可发布图文素材包的正文终稿必须调用 `/Users/wangbin/.codex/skills/humanizer/SKILL.md` 润色；不要只参考规则后自行改写。
- 证据写入 `04_分析依据/humanizer润色记录.md`，至少包含：润色前草稿摘要、humanizer 发现的 AI 感问题、HKR 判断、价值感自查、最终正文、确认正文无话题标签。
- `publish_manifest.json` 必须写入 `humanizer_used: true`，并写入 `humanizer_record` 指向记录文件。

## Imagegen 规则

- 需要新封面或新配图时，必须调用 `/Users/wangbin/.codex/skills/.system/imagegen/SKILL.md` 生成最终视觉来源。
- 不允许用 PIL、matplotlib、HTML/CSS、SVG、canvas 或截图式代码图替代 imagegen 作为最终发布图来源。
- 本地代码只允许做裁剪、压缩、拆图、搬运、加水印和 manifest/Excel 适配。
- 有爆款图片、表格截图、代码截图、数据结果图或场景图时，优先把参考图作为事实来源和构图参考，通过 imagegen 重绘或重排；清理账号名、头像、水印、私信入口、二维码、联系方式和未脱敏隐私。
- 最终发布图不能出现内部流程词、文件路径、prompt、TODO、manifest、校验词、模型名、工具名或平台限制表。
- 证据写入 `04_分析依据/imagegen生成记录.md`。逐图记录：用途、参考图路径或跳过原因、保留的信息点、二创动作、清理项、最终上传图路径、人工检查结果。
- `publish_manifest.json` 必须写入 `imagegen_used: true`、`imagegen_record`、`image_generation_method`；如果明确只用已有图片，必须写入 `imagegen_skipped_reason`。

## 爆款学习与重复判断

- 账号数据优先。外部爆款必须做“1:1 参考式模仿”：贴着学习标题公式、钩子位置、事实选择、场景设置、案例组织、开头方式、正文骨架、段落节奏、页面顺序、图片素材思路、封面逻辑、价值呈现、共鸣写法和评论触发方式。这里的 1:1 指结构、节奏、信息组织和阅读路径要高度对齐，不是泛泛总结。可以保留可验证事实、通用场景、问题结构和情绪转折，但必须换成自己的表达，不复制原句、独特表达、作者身份设定、头像、账号名、私有数据或专有作品。第一人称内容只能改写为模拟场景、读者场景或公开可验证案例，不冒充他人的真实亲历。
- 只有标题、互动数或媒体、缺少 `note_text`/`推文`/`正文` 的样本，只能学习标题和封面入口，不能学习正文写法。
- 同一主题可以连续生产不同角度内容；历史发布记录用于避开重复表达，不用于直接否决新内容。
- 只有同一 `publish_manifest.json`、同一发布 key、标题正文高度一致且图片组合基本一致时，才判定为同一篇重复内容并停止；停止原因写成“同一篇重复提交”。
- 如果已有同主题内容，新素材包至少替换 3 项：读者问题、切入角度、字段清单、流程步骤、案例场景、封面钩子、图片结构或素材来源。

## 发布前共同检查

- 标题 <= 20 字，正文 <= 1000 字，图片 <= 18 张。
- 正文无话题标签，话题在独立字段。
- 存在 `04_分析依据/humanizer润色记录.md`，manifest 中 `humanizer_used` 为 `true`。
- 存在 `04_分析依据/imagegen生成记录.md`；manifest 中 `imagegen_used` 为 `true`，或存在明确的 `imagegen_skipped_reason`。
- 上传图均存在，且不是未拆分的拼接图或内部占位图。
- 标题、正文、封面字、配图字、话题、合集名无明显历史违规风险。
